API Overview and Documentation

API Overview

Welcome to the Remium API documentation. This API provides programmatic access to Remium's banking and financial services platform.

Base URL

All API requests should be made to the appropriate environment base URL:

  • Production: \https://gw.remium.uk\
  • Sandbox: \https://gw.bastionresearch.com\

Authentication

The Remium API uses OAuth 2.0 Bearer token authentication.

For every request, you must include the 'Authorization' header with the value 'Bearer {accessToken}'.

You can obtain an access token using the /login-service/api/v1/sign-in/email/password\ endpoint described below, by providing your login credentials. The access token is valid for 24 hours, after which you need to request a new one.

Authentication Header Example

Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

Token Endpoints

  • Get Access Token: \POST /login-service/api/v1/sign-in/email/password\
  • Refresh Token: \POST /login-service/api/v1/auth/refresh\

Request Format

All API requests should include:

  • 'Content-Type: application/json' header
  • Properly formatted JSON request bodies
  • 'Authorization: Bearer {accessToken}' header for authenticated endpoints

Error Handling

The API uses standard HTTP status codes and provides detailed error messages:

  • 400 Bad Request: Invalid request parameters
  • 401 Unauthorized: Authentication required or invalid
  • 403 Forbidden: Insufficient permissions
  • 404 Not Found: Resource not found
  • 500 Internal Server Error: Server-side error

Webhooks

Remium supports webhooks for real-time event notifications. Configure your webhook endpoints to receive notifications for events like:

  • Client onboarding status changes
  • Account opened

Webhook Events and Payloads

The Remium system sends the following webhook events:

Client Approved

Sent when a client's application has been successfully approved and verified.

{
  "type": "ClientVerified",
  "data": {
    "clientId": "cli-JohnSmith.remium.VNIsaEjWIL13",
    "clientName": "John Smith",
    "state": "verified"
  }
}

Client Rejected

Sent when a client's application has been declined or rejected.

{
  "type": "ClientRejected",
  "data": {
    "clientId": "cli-JohnSmith.remium.VNIsaEjWIL13",
    "clientName": "John Smith",
    "state": "declined"
  }
}

Account Opened

Sent when a new account has been successfully opened for a client.

{
  "type": "AccountOpened",
  "data": {
    "accountId": "acc.USD.TLfBCnWnwbEwf34",
    "clientId": "cli-JohnSmith.remium.VNIsaEjWIL13",
    "currencyId": "USD",
    "number": "SDJ-MXJJJ4421846-Q41"
  }
}

Retry Policy

Failed webhook deliveries will be retried with exponential backoff for up to 24 hours. Ensure your endpoint returns a 2xx status code to acknowledge successful receipt.
